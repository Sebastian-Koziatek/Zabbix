
<h2 align="center">Instalacja Grafana OSS na Ubuntu/Debian</h2>

Grafana OSS (Open Source) to najpopularniejsza wersja platformy do wizualizacji danych. Zalecana metoda instalacji to użycie oficjalnego repozytorium APT, ale w przypadku problemów z repozytorium można skorzystać z instalacji przez plik .deb.

---

## Metoda 1: Instalacja przez repozytorium APT (zalecana)

1. Instalacja pakietów wstępnych:
	```bash
	sudo apt-get install -y apt-transport-https software-properties-common wget
	```
2. Import klucza GPG:
	```bash
	sudo mkdir -p /etc/apt/keyrings/
	wget -q -O - https://apt.grafana.com/gpg.key | gpg --dearmor | sudo tee /etc/apt/keyrings/grafana.gpg > /dev/null
	```
3. Dodanie repozytorium Grafana OSS (stable):
	```bash
	echo "deb [signed-by=/etc/apt/keyrings/grafana.gpg] https://apt.grafana.com stable main" | sudo tee -a /etc/apt/sources.list.d/grafana.list
	```
4. Aktualizacja listy pakietów:
	```bash
	sudo apt-get update
	```
5. Instalacja Grafana OSS:
	```bash
	sudo apt-get install grafana
	```
6. Uruchomienie i włączenie usługi Grafana:
	```bash
	sudo systemctl start grafana-server
	sudo systemctl enable grafana-server
	```

---

## Metoda 2: Instalacja przez plik .deb (gdy repozytorium APT nie działa)

1. Instalacja wymaganej zależności:
	```bash
	sudo apt-get install musl
	```
2. Pobranie najnowszego pakietu Grafana OSS:
	```bash
	wget https://dl.grafana.com/oss/release/grafana_10.4.2_amd64.deb
	```
3. Instalacja pakietu:
	```bash
	sudo dpkg -i grafana_10.4.2_amd64.deb
	```
4. Uruchomienie i włączenie usługi Grafana:
	```bash
	sudo systemctl enable grafana-server
	sudo systemctl start grafana-server
	```

**Uwaga:** Jeśli pojawią się błędy zależności lub brak pliku usługi, sprawdź czy pakiet musl jest dostępny w Twojej dystrybucji. Zalecane jest korzystanie z repozytorium APT, które automatycznie rozwiązuje zależności.

---

## Troubleshooting

- Jeśli pojawia się błąd "Temporary failure resolving 'apt.grafana.com'", sprawdź konfigurację DNS w pliku `/etc/resolv.conf` i dodaj np.:
  ```
  nameserver 8.8.8.8
  nameserver 1.1.1.1
  ```
- Jeśli instalacja przez .deb zgłasza brak zależności lub pliku usługi, usuń nieudaną instalację:
  ```bash
  sudo dpkg --remove grafana
  ```
  i spróbuj ponownie przez repozytorium APT lub Docker.
- Alternatywnie możesz uruchomić Grafanę przez Docker:
  ```bash
  sudo apt-get install docker.io
  sudo docker run -d -p 3000:3000 --name=grafana grafana/grafana-oss
  ```
